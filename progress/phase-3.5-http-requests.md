## Notes

- Direction values: `inbound` (requests arriving at the app) | `outbound` (requests sent to external services)
- Slow threshold default: 1000ms — requests exceeding this get `is_slow = 1` at ingest time
- `trace_id` is generated by the SDK per inbound request and propagated to all outbound calls within the same lifecycle
- Error linkage is read-only on this endpoint — errors table is written by Phase 2 pipeline
- ClickHouse TTL: 30 days retention (same as errors)
- Data flow: SDK → POST /ingest/v1/http-requests → Kafka (allstak.http_requests) → HttpRequestKafkaConsumer → ClickHouse (http_requests)
