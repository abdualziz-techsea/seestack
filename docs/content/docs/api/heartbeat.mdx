      - content: |
          Signals the completion of a cron job or scheduled task.
          Call this at the **end** of every job execution, regardless of
          success or failure.

          - `slug` must match an existing cron monitor created in the
            SeeStack management console
          - If `status` is `"failed"`, alerts are evaluated immediately
          - `durationMs` is used for trend analysis

          **Note:** This endpoint returns a raw JSON object, not the standard
          `ApiResponse` envelope.

          **SDK integration pattern:**
          ```
          start = now()
          try {
            runJob()
            sendHeartbeat(slug, "success", now() - start, null)
          } catch (e) {
            sendHeartbeat(slug, "failed", now() - start, e.message)
            throw e  // rethrow — heartbeat must not swallow job errors
          }
          ```
